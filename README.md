# go-distributed-systems

Documentation, notes and journey of learning production-focused backend systems built with Go, Kafka, Postgres, and AWS.

---

## Objective

Build a strong foundation in designing and implementing scalable, fault-tolerant distributed systems using Go and modern backend infrastructure.

---

## Tech Stack

- Go (core language)
- Kafka (event streaming, async communication)
- PostgreSQL (data storage, transactions)
- AWS (deployment, scaling, infra)
- Docker (containerization)

---

## Learning Approach

- First principles understanding (why before how)
- Build small, focused examples per concept
- Gradually move to real-world systems
- Emphasis on production concerns:
  - reliability
  - scalability
  - observability
  - fault tolerance

---

## Goal

Move from learning concepts → building production-grade systems with clear understanding of trade-offs and system behavior under load.
